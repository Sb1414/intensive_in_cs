CheckList. Team01

Exercise 00
Exercise 00 - Basics

- Are the WeatherForecast, WeatherForecastController, Startup, Program, appsettings, appsettings.Development files present in the project?
- Does the Get() method in WeatherForecastController have XML-comments with a description of the method?
- Does the swagger-documentation of the application open and is it possible to read the description of the Get() method on its page?

Exercise 01
Exercise 01 - Environment.

- If you run the application in a Production environment, the swagger-documentation should not be unavailable. Is it so?

Exercise 02
Exercise 02 - GET weather by coordinates.

- Does the Get() method use a remote resource to get a weather forecast?
- Is it possible to call it from the browser and get the weather forecast? Will the result change when the coordinates change?
- Is information about temperature, pressure, humidity, wind speed, the name of a geographical object and a description of the weather output?

Exercise 03
Exercise 03 - GET weather by city.


- Does the method use a remote resource to get a weather forecast?
- Is it possible to call it from the browser using the link /WeatherForecast /{cityName} and get the weather forecast? Will the result change when the city name is changed?
- Is information about temperature, pressure, humidity, wind speed, the name of a geographical object and a description of the weather output?

Exercise 04
Exercise 04 - Code refactoring.


- Is the solution divided into 2 projects? Does the rush01.WeatherClient project have a service responsible for communicating with a remote resource? Does it contain 2 separate methods?
- Is the API KEY value moved to the appsettings.json and appsettings.Development.json files? Is it passed to the WeatherClient class and used as an argument?
- Is an object of the type WeatherClient private and readonly a property of the controller? Is it passed to the constructor to initialize the property?
- Is WeatherClient registration configured in Startup?
- Does the WeatherClient class accept an object of the IOptions type in the constructor? Is it used to set the key for connecting to a remote resource?

Exercise 05
Exercise 05 - POST/PUT/DELETE, repeat.


- Is the Post() method implemented in the controller, which takes a text value and writes it to the cache?
- Is it true:
- Call the Get method without specifying the city. The method should return the status 404.
- Call the Get method specifying the city of Barnaul. The method should return the weather for the city of Barnaul.
- Call the Post method specifying the city of Albuquerque. The method should return the status 200.
- Call the Get method specifying the city of Barnaul. The method should return the weather for the city of Barnaul.
- Call the Get method without specifying the city. The method should return the weather for the city of Albuquerque.
